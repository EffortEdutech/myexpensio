1. Auth & onboarding
myexpensio uses invite-only onboarding. There is no public sign-up. All users must be invited by an admin of their organisation.
Onboarding type
Invite-only. No public registration.
Who can invite
Admin users only, via the admin app.
Invite validity
Token expires after set period. One-time use.
Organisation scope
Each user belongs to one org. All data is org-scoped.
User roles
RoleAccess levelApp
UserOwn trips, claims, exportsUser app
Admin AdminAll users in org, invite management, rates, templatesAdmin app
If your invite link is expired or already used, contact your admin to request a new one.

3. Claims & items
A claim is a submission covering a period. It contains one or more items of different types. Claims move from Draft to Submitted — and once submitted, they are locked permanently in Phase 1.
Claim lifecycle
StatusEditable?Description
DraftYesBeing built. Items can be added, edited, or removed.
Submitted LockedNoLocked permanently. No edits, no deletes. Values are frozen for audit.
Once submitted, a claim cannot be edited. This is a hard system rule — not a UI restriction. Always review all items before submitting.

4. Transport claims
Toll, Parking, and Bus items capture non-mileage transport costs. Each supports both cash and TNG eWallet as payment methods.
Toll
Enter amount + payment method. Optional: import from TNG statement.
Parking
Enter amount, location, duration. Attach receipt optionally.
Bus fare
Enter fare, origin/destination stops. Payment: cash or TNG.
Payment methods
Cash or TNG eWallet. Recorded per transaction.
5. TNG statement import
Upload your Touch 'n Go eWallet PDF statement to automatically extract toll and parking transactions for import into a claim.
Import flow
StepAction
1In a Toll or Parking item, tap "Add from TNG Statement".
2Upload your TNG eWallet PDF statement.
3The app parses the statement and shows a transaction picker.
4Select the relevant transactions. They are imported as claim items.
The parser uses transaction reference numbers (Trans No) to reliably identify rows. Multi-page statements are supported.
6. Exports
Exports produce a CSV or XLSX file of your submitted claims. Values are deterministic — they reflect what was submitted and cannot differ from the locked claim record.
Formats
CSV and XLSX.
Default status filter
Submitted only (recommended for finance).
Distance in export
KM, 2 decimal places.
Amounts in export
MYR. Deterministic — locked to submitted values.
Never recalculate reimbursement amounts after submission. Exports always use the submitted snapshot — this protects you in finance disputes.
7. Subscription tiers
myexpensio has two tiers in Phase 1. The key limit is route calculations per month — GPS tracking is always unlimited.

8. Admin responsibilities
Admins manage the organisation via the separate admin app. Users in the user app cannot perform admin actions.
ResponsibilityDescription
Invite managementCreate and revoke invite links for new users.
Rate configurationSet org-wide mileage, meal, and lodging default rates.
Per diem settingsSet org-wide daily per diem rate. Users can override per claim.
Export templatesConfigure which columns appear in exports and in what order.
Claim reviewReview submitted claims from all org members.
The admin app is a separate interface. If you need admin access, contact your organisation administrator.
