# Scan Service — Deployment Runbook
**Service:** myexpensio-scan (Python FastAPI + OpenCV)  
**Host:** Render.com free tier, Singapore region  
**Warmup:** Vercel cron every 14 minutes → Render /health

---

## Architecture

```
User phone
    ↓
Vercel (Next.js)  ←── cron every 14min keeps Render awake
    ↓  /api/scan/process  (secret never exposed to browser)
Render.com (Python + OpenCV)
    ↓  processed JPEG base64
Vercel → phone
```

---

## Step 1 — Push scan_service to GitHub

scan_service/ folder must be in the root of your myexpensio repo.

```
myexpensio/
├── apps/
│   └── user/
├── scan_service/        ← this folder
│   ├── main.py
│   ├── requirements.txt
│   └── render.yaml
└── ...
```

```powershell
git add scan_service/
git add "apps/user/app/api/scan/process/route.ts"
git add "apps/user/app/api/cron/warmup/route.ts"
git add apps/user/components/ScanPreviewModal.tsx
git add apps/user/vercel.json
git commit -m "feat: Python OpenCV scan service + warmup cron"
git push
```

---

## Step 2 — Deploy to Render.com

1. Go to https://render.com → Sign in (free account)
2. New → **Web Service**
3. Connect GitHub → select **myexpensio** repo
4. Fill in:
   - **Name:** `myexpensio-scan`
   - **Root Directory:** `scan_service`
   - **Runtime:** Python
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `uvicorn main:app --host 0.0.0.0 --port $PORT`
   - **Region:** Singapore
   - **Plan:** Free
5. Click **Create Web Service**
6. Wait for first deploy (~3 minutes)
7. Your URL will be: `https://myexpensio-scan.onrender.com`

---

## Step 3 — Get the SCAN_API_SECRET from Render

1. Render dashboard → myexpensio-scan → **Environment**
2. Find `SCAN_API_SECRET` (auto-generated by render.yaml)
3. Click **Reveal** → copy the value

---

## Step 4 — Add env vars to Vercel

Vercel dashboard → myexpensio project → Settings → Environment Variables

Add these 3 vars (all environments: Production + Preview + Development):

| Key | Value |
|-----|-------|
| `SCAN_API_URL` | `https://myexpensio-scan.onrender.com` |
| `SCAN_API_SECRET` | (paste from Render step 3) |
| `CRON_SECRET` | (generate any random string, e.g. `openssl rand -hex 32`) |

Then **redeploy** Vercel so new env vars take effect.

---

## Step 5 — Add env vars to .env.local (local dev)

```
# apps/user/.env.local  — add these lines

# For local dev: run Python locally on port 8000
SCAN_API_URL=http://localhost:8000
SCAN_API_SECRET=dev_secret_local
CRON_SECRET=dev_cron_secret_local
```

## Step 6 — Run Python locally for development

```powershell
# In a new PowerShell window
cd "C:\Users\user\Documents\00 Reimbursement Assistant\myexpensio\scan_service"

# First time only — install deps
pip install -r requirements.txt

# Start server
uvicorn main:app --reload --port 8000
```

Verify it works:
```powershell
# Should return {"status":"ok","service":"myexpensio-scan"}
Invoke-WebRequest -Uri "http://localhost:8000/health" | Select-Object Content
```

After deploying to Vercel:
1. Vercel dashboard → myexpensio → **Cron Jobs** tab
2. You should see `/api/cron/warmup` scheduled every 14 minutes
3. Click **Run Now** to test manually

## Cold start behaviour (free tier)

If Render service was asleep (warmup cron missed or first deploy):
- First enhance request takes 30–50 seconds
- ScanPreviewModal shows "Enhancing…" spinner throughout
- Subsequent requests that session are instant (~1–2 seconds)

Warmup cron prevents this in normal operation.
